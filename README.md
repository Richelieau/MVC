# MVC
Model-View-Controller (MVC) adalah pola arsitektur perangkat lunak yang memisahkan aplikasi menjadi tiga komponen utama untuk mempermudah pengembangan dan pemeliharaan kode. 
Berikut adalah penjelasan fungsi dari masing-masing komponen:
Model (Data & Logika Bisnis)
Bagian ini bertanggung jawab mengelola data, database, dan aturan bisnis aplikasi. Model akan berinteraksi langsung dengan database untuk mengambil, menyimpan, atau memperbarui data.
View (Tampilan)
Bagian yang dilihat oleh pengguna. View berfungsi menyajikan informasi dalam bentuk antarmuka visual (UI). Ia tidak memiliki logika proses sendiri, melainkan hanya menampilkan data yang dikirimkan oleh Controller.
Controller (Penghubung)
Otak dari aplikasi yang menerima masukan (input) dari pengguna (seperti klik atau ketikan). Controller bertugas memproses permintaan tersebut, berinteraksi dengan Model, lalu memilih View mana yang harus ditampilkan kepada pengguna.
